## User Story

**Como** gestante,
**quero** cadastrar minhas consultas pré-natais com data, hora e tipo,
**para que** o app me lembre com antecedência e eu não perca nenhuma consulta.

---

## Acceptance Criteria

- [ ] **AC1** — Nova tela/seção "Consultas Pré-Natais" acessível via aba Ferramentas
- [ ] **AC2** — CRUD completo: criar, listar, editar, deletar consultas
- [ ] **AC3** — Campos: tipo (Obstetra / Ultrassom / Exames / Outro), data, hora, observação (opcional)
- [ ] **AC4** — Lista ordenada por data, próximas consultas primeiro
- [ ] **AC5** — Persistência em SQLite via nova tabela `prenatal_appointments` (criada via `runMigrations()`)
- [ ] **AC6** — Schema: `prenatal_appointments(id INTEGER PRIMARY KEY, type TEXT, appointment_date TEXT, appointment_time TEXT, notes TEXT, reminder_offset TEXT, created_at TEXT)`
- [ ] **AC7** — Lembrete configurável: `reminder_offset` pode ser "1day", "2hours" (padrão), "ontime"
- [ ] **AC8** — Ao criar/editar consulta: agendar notificação via `useNotifications` (Story 3.2)
- [ ] **AC9** — Ao deletar consulta: cancelar notificação correspondente pelo ID
- [ ] **AC10** — `npm run typecheck` zero erros
