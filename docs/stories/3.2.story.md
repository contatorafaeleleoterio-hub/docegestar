## User Story

**Como** gestante,
**quero** que o app solicite permissão para enviar notificações e as agende automaticamente,
**para que** eu seja lembrada sem precisar abrir o app.

---

## Acceptance Criteria

- [ ] **AC1** — `expo-notifications` instalado e configurado (`npx expo install expo-notifications`)
- [ ] **AC2** — Permissão solicitada na primeira vez que usuária ativa um lembrete (não no onboarding)
- [ ] **AC3** — Fallback gracioso se permissão negada: toggles ficam desativados, UI exibe "Permissão necessária para notificações"
- [ ] **AC4** — Notificações agendadas com ID único por tipo (permite cancelamento/reagendamento)
- [ ] **AC5** — Ao abrir o app: verificar se notificações vencidas precisam ser reagendadas
- [ ] **AC6** — Ao editar configurações: cancelar notificações antigas e reagendar com novos parâmetros
- [ ] **AC7** — Novo hook `src/hooks/useNotifications.ts` encapsula toda a lógica de permissão e agendamento
- [ ] **AC8** — Plataforma web: `expo-notifications` não funciona — toda lógica de notificação desabilitada graciosamente com `Platform.OS === 'web'` guard
- [ ] **AC9** — `npm run typecheck` zero erros
